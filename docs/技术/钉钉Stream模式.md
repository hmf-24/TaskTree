# 钉钉Stream模式配置指南

## 概述

钉钉Stream模式允许机器人通过WebSocket主动连接钉钉服务器，无需公网IP即可接收和响应用户消息。这对于本地开发环境特别有用。

## 配置步骤

### 1. 获取钉钉应用凭证

1. 访问 [钉钉开放平台](https://open-dev.dingtalk.com/)
2. 创建或选择一个企业内部应用
3. 在应用详情页面找到：
   - **AppKey** (Client ID)
   - **AppSecret** (Client Secret)

### 2. 配置机器人权限

确保应用具有以下权限：
- 机器人接收消息
- 发送消息到企业群

### 3. 配置环境变量

在 `.env` 文件中添加以下配置：

```bash
# 启用钉钉Stream模式
DINGTALK_STREAM_ENABLED=true

# 钉钉应用凭证
DINGTALK_CLIENT_ID=your_app_key_here
DINGTALK_CLIENT_SECRET=your_app_secret_here
```

### 4. 重启应用

```bash
docker-compose restart backend
```

### 5. 验证启动

查看后端日志，应该看到：

```
🚀 正在启动钉钉Stream客户端...
   ClientID: dingxxxxx...
✅ 钉钉Stream客户端已启动
🎉 钉钉Stream模式已启动
```

## 使用方法

### 1. 绑定钉钉账号

1. 登录TaskTree系统
2. 进入"设置"页面
3. 在"智能提醒"标签页中配置钉钉Webhook（用于发送消息）
4. 保存设置

### 2. 与机器人交互

在钉钉群聊中@机器人，发送以下类型的消息：

#### 查询任务详情
```
@机器人 登录功能的详细情况
```

#### 更新任务进度
```
@机器人 登录功能完成了80%
```

#### 报告问题
```
@机器人 登录功能遇到了数据库连接问题
```

#### 延期任务
```
@机器人 登录功能需要延期3天
```

## 工作原理

```
┌─────────────┐         WebSocket         ┌──────────────┐
│   TaskTree  │ ◄────────────────────────► │ 钉钉服务器   │
│  (本地环境)  │                            │              │
└─────────────┘                            └──────────────┘
      ▲                                            │
      │                                            │
      │ 处理消息                                    │ 推送消息
      │                                            │
      ▼                                            ▼
┌─────────────┐                            ┌──────────────┐
│ 消息处理器   │                            │   用户设备   │
│ (复用现有)   │                            │  (钉钉客户端) │
└─────────────┘                            └──────────────┘
```

## Stream模式 vs Webhook模式

| 特性 | Stream模式 | Webhook模式 |
|------|-----------|------------|
| 公网IP | ❌ 不需要 | ✅ 需要 |
| 域名 | ❌ 不需要 | ✅ 需要 |
| 本地开发 | ✅ 支持 | ❌ 不支持 |
| 连接方式 | 主动连接 | 被动接收 |
| 消息延迟 | 低 | 低 |
| 部署复杂度 | 简单 | 复杂 |

## 故障排查

### 问题1: 启动时提示"配置不完整"

**原因**: 未配置 `DINGTALK_CLIENT_ID` 或 `DINGTALK_CLIENT_SECRET`

**解决**: 检查 `.env` 文件，确保配置正确

### 问题2: @机器人后无响应

**可能原因**:
1. 用户未绑定钉钉账号
2. Stream客户端未启动
3. 网络连接问题

**解决步骤**:
1. 查看后端日志确认Stream客户端已启动
2. 在系统设置中绑定钉钉账号
3. 检查网络连接

### 问题3: 连接频繁断开

**原因**: 网络不稳定

**解决**: SDK会自动重连，等待几秒钟即可恢复

## 安全建议

1. **不要泄露凭证**: `AppKey` 和 `AppSecret` 是敏感信息，不要提交到代码仓库
2. **使用环境变量**: 通过 `.env` 文件配置，不要硬编码
3. **定期更新**: 定期更新 `AppSecret` 以提高安全性
4. **权限最小化**: 只授予机器人必需的权限
