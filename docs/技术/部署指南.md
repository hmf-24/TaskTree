# TaskTree 部署指南

本文档详细介绍 TaskTree 项目的各种部署方式，包括开发环境、生产环境 Docker 部署等。

---

## 环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Docker | 20.10+ | 容器引擎 |
| Docker Compose | 2.0+ | 容器编排 |
| Node.js | 18+ | 本地开发 |
| Python | 3.11+ | 本地开发 |

---

## 快速开始（Docker 推荐）

```bash
# 1. 进入项目目录
cd TaskTree

# 2. 复制环境配置
copy .env.example .env

# 3. 启动所有服务
docker-compose up -d --build

# 4. 查看运行状态
docker-compose ps

# 5. 查看日志（可选）
docker-compose logs -f
```

服务启动后访问：**http://localhost**

---

## 详细部署步骤

### 1. 环境配置

#### 1.1 配置环境变量

复制环境配置模板：

```bash
# Windows
copy .env.example .env

# Linux/Mac
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# 后端配置
SECRET_KEY=your-secret-key-change-in-production
DEBUG=false

# 数据库 (SQLite 文件路径)
DATABASE_URL=sqlite+aiosqlite:///./data/tasktree.db
```

> **注意**：生产环境请修改 `SECRET_KEY` 为强密钥，建议使用随机生成的字符串。

#### 1.2 拉取基础镜像（可选，加速首次构建）

```bash
docker pull python:3.11-slim
docker pull node:20-alpine
docker pull nginx:alpine
```

### 2. Docker Compose 配置

#### 2.1 服务架构

```
┌─────────────────────────────────────────┐
│            Docker Host                  │
│  ┌──────────────────┐  ┌───────────┐ │
│  │   Frontend      │  │  Backend  │ │
│  │   (Nginx:80)    │  │ (uvicorn) │ │
│  │   localhost     │  │  :8000    │ │
│  └────────┬─────────┘  └─────┬─────┘ │
│           │                  │        │
│           └────────┬─────────┘        │
│                    │                 │
│              ┌────▼────┐           │
│              │ SQLite │            │
│              │  Data  │            │
│              └────────┘            │
└─────────────────────────────────────────┘
```

#### 2.2 端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端 | 80 | Nginx HTTP 端口 |
| 后端 | 8000 | FastAPI 端口 |

### 3. 启动服务

```bash
# 首次构建并启动
docker-compose up -d --build

# 启动（已有镜像）
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f backend
docker-compose logs -f frontend
```

### 4. 验证部署

```bash
# 检查后端健康
curl http://localhost:8000/api/v1/tasktree/auth/me

# 检查前端
curl http://localhost
```

## 本地开发模式

### 后端启动

```bash
# 1. 进入后端目录
cd backend

# 2. 创建虚拟环境
python -m venv venv

# 3. 激活虚拟环境
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# 4. 安装依赖
pip install -r requirements.txt

# 5. 初始化数据库
python -c "from app.core.database import init_db; init_db()"

# 6. 启动后端
uvicorn app.main:app --reload --port 8000 --host 0.0.0.0
```

访问：http://localhost:8000/docs （API 文档）

### 前端启动

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

访问：http://localhost:5173

### 前端构建

```bash
# 1. 安装依赖
npm install

# 2. 构建生产版本
npm run build

# 3. 预览构建结果（可选）
npm run preview
```

## 生产构建

### 方式一：Docker 构建

```bash
# 构建所有服务
docker-compose build

# 启动生产服务
docker-compose up -d
```

### 方式二：手动构建

```bash
# 1. 前端构建
cd frontend
npm install
npm run build

# 2. 后端构建
cd ../backend
pip install -r requirements.txt

# 3. 启动后端
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## 常用命令参考

### Docker Compose

```bash
# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f

# 查看状态
docker-compose ps

# 进入容器（调试）
docker-compose exec backend bash
docker-compose exec frontend sh
```

### Docker 清理

```bash
# 停止并删除容器
docker-compose down -v

# 删除未使用的镜像
docker image prune -f

# 删除未使用的数据卷
docker volume prune -f

# 完全清理（谨慎使用）
docker system prune -af
```

## 数据管理

### 数据库文件位置

- SQLite 数据库：`backend/data/tasktree.db`
- 数据卷：`tasktree_backend-data`

### 备份数据库

```bash
# 复制数据库文件
docker cp tasktree-backend:/app/data/tasktree.db ./backup.db
```

### 恢复数据库

```bash
# 停止服务
docker-compose down

# 复制数据库文件
docker cp ./backup.db tasktree-backend:/app/data/tasktree.db

# 启动服务
docker-compose up -d
```

### 重置数据

```bash
# 停止服务
docker-compose down -v

# 删除数据卷
docker volume rm tasktree_backend-data

# 重新初始化
docker-compose up -d --build
```

## 故障排查

### 常见问题

#### 1. 端口被占用

```bash
# 检查端口占用
netstat -ano | findstr :80
netstat -ano | findstr :8000

# 结束占用进程
taskkill /PID <进程ID> /F
```

#### 2. 数据库锁定

```bash
# 删除锁文件后重试
docker-compose down
docker volume rm tasktree_backend-data
docker-compose up -d
```

#### 3. 内存不足

```bash
# 调整 Docker 内存限制
# Docker Desktop -> Settings -> Resources -> Memory
```

#### 4. 构建失败

```bash
# 清理 Docker 缓存后重试
docker builder prune
docker-compose build --no-cache
```

---

## 安全建议

1. **修改默认密钥**：生产环境务必修改 `SECRET_KEY`
2. **限制端口**：只对外开放必要端口（80, 443）
3. **定期更新**：关注安全更新，及时升级基础镜像
4. **数据备份**：定期备份数据库文件
5. **日志监控**：配置日志收集和告警
