## 已实现功能 ✅

| 功能 | 状态 | 说明 |
| :--- | :--- | :--- |
| API Key加密存储 | ✅ | 使用crypto.py加密，字段llm_api_key_encrypted |
| 消息去重 | ✅ | 检查最近6小时，跳过已发送任务 |
| 调度状态持久化 | ✅ | SchedulerState表记录last_run_at |
| 手动触发不占配额 | ✅ | is_manual区分 |
| 多厂商大模型 | ✅ | Minimax/OpenAI/Anthropic |
| 多维度分析 | ✅ | 逾期/进度/依赖/负荷/复杂度/风险 |
| 统计报表 | ✅ | 区分手动/自动发送 |

## 配额逻辑

| 触发方式 | 是否计入配额 | 去重窗口 |
| :--- | :--- | :--- |
| 自动调度 | ✅ 是 | 每日上线(daily_limit) |
| 手动触发 | ❌ 否 | 6小时 |

---

## 加密说明

> [!NOTE]
> 加密算法：base64 + SHA256 XOR混淆，如有cryptography库自动升级为Fernet(AES)

- 密钥来源：环境变量 `ENCRYPTION_KEY` 或应用默认密钥
- 建议生产环境设置 `ENCRYPTION_KEY` 环境变量
