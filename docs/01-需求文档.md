# 任务树（TaskTree）需求文档 v1.0

## 1. 文档概述

### 1.1 文档目的

本文档详细描述任务树（TaskTree）的功能需求、非功能需求、技术架构和开发计划，为开发团队提供明确的开发依据和验收标准。

### 1.2 产品名称

TaskTree - 直观高效的任务分解与项目管理工具

### 1.3 目标用户

- 个人开发者和自由职业者

- 小型团队（3-10人）

- 产品经理、项目经理

- 学生和教育工作者

### 1.4 文档范围

本文档覆盖TaskTree v1.0版本的所有功能需求和技术要求，不包含后续版本的规划内容。

## 2. 产品愿景与目标

### 2.1 产品愿景

打造一款"开箱即用"的任务分解工具，让复杂项目变得简单清晰，帮助用户从"无从下手"到"步步为营"。

### 2.2 核心目标

1. **极致简单**：无需培训，5分钟内上手使用

2. **无限层级**：支持任意深度的任务分解

3. **直观操作**：拖拽式界面，所见即所得

4. **轻量高效**：专注核心功能，不做臃肿的项目管理系统

### 2.3 成功指标

- 单个任务分解操作耗时<3秒
- 页面加载时间<1秒
- 核心功能用户完成率>95%
- 数据导出成功率100%
- 跨平台：Web + Windows (Docker)

## 3. 用户角色与典型场景

### 3.1 用户角色

|角色|描述|核心需求|
|---|---|---|
|个人用户|管理个人项目和待办事项|快速分解任务、进度跟踪、提醒|
|团队负责人|分配任务给团队成员|任务分配、进度监控、协作沟通|
|开发者|执行具体开发任务|任务详情、依赖关系、时间估算|
|产品经理|规划产品功能迭代|需求分解、优先级排序、版本管理|
### 3.2 典型场景

**场景1：个人项目启动**

小明要开发一个个人网站，他打开TaskTree，创建"个人网站开发"项目，然后分解为：需求分析、UI设计、前端开发、后端开发、测试上线。每个大任务再继续分解为更小的子任务，直到每个任务都可以在1-2天内完成。

**场景2：团队任务分配**

张经理带领5人团队开发一个新功能，他先在TaskTree中完成整体任务分解，然后将不同的任务卡片拖拽分配给对应的团队成员，设置截止日期和依赖关系。团队成员可以实时看到自己的任务和整体进度。

**场景3：项目进度跟踪**

李产品每天早上打开TaskTree，查看项目整体进度和各个任务的完成情况，识别延期风险，及时调整计划。

## 4. 功能需求

### 4.1 核心功能模块

#### 4.1.1 项目管理

- ✅ **创建项目**：输入项目名称、描述、开始/结束日期

- ✅ **编辑项目**：修改项目基本信息

- ✅ **删除项目**：支持软删除和永久删除

- ✅ **项目列表**：显示所有项目，支持按名称、创建时间、进度排序

- ✅ **项目归档**：完成的项目可以归档，不显示在主列表中

#### 4.1.2 任务分解（核心中的核心）

- ✅ **无限层级分解**：支持任意深度的任务嵌套（任务→子任务→子子任务...）

- ✅ **快速创建任务**：点击"+"按钮或按Enter键快速创建同级任务

- ✅ **创建子任务**：按Tab键或点击"添加子任务"按钮创建下一级任务

- ✅ **任务重命名**：双击任务名称直接编辑

- ✅ **拖拽调整**：

    - 上下拖拽调整任务顺序

    - 左右拖拽调整任务层级

    - 拖拽任务到另一个父任务下

- ✅ **任务删除**：删除任务时提示是否同时删除所有子任务

#### 4.1.3 任务卡片详情

- ✅ **基本信息**：任务名称、描述、负责人、截止日期

- ✅ **优先级**：高/中/低三个优先级，用不同颜色标识

- ✅ **状态**：待办/进行中/已完成/已取消

- ✅ **进度**：0-100%进度条，子任务进度自动汇总到父任务

- ✅ **标签**：支持自定义标签，用于分类和筛选

- ✅ **时间估算**：记录预计耗时和实际耗时

- ✅ **附件**：支持上传图片、文档等附件（v1.0 表结构已预留，API 待开发）

- ✅ **评论**：添加评论和讨论，支持@提及团队成员

#### 4.1.4 依赖关系管理

- ✅ **创建依赖**：拖拽任务之间的连线创建依赖关系（A完成后B才能开始）

- ✅ **可视化显示**：用箭头清晰显示任务之间的依赖关系

- ✅ **依赖检查**：尝试开始有未完成前置任务的任务时给出提示

- ✅ **删除依赖**：点击连线删除依赖关系

#### 4.1.5 视图切换

- ✅ **大纲视图**：默认视图，树形结构显示所有任务

- ✅ **看板视图**：按状态分组显示任务卡片（待办/进行中/已完成）

- ✅ **甘特图视图**：时间线形式显示任务进度和依赖关系

- ✅ **列表视图**：简洁的列表形式，适合快速浏览

#### 4.1.6 团队协作

- ✅ **成员邀请**：通过邮箱邀请团队成员加入项目

- ✅ **权限管理**：

    - 所有者：完全控制权限

    - 编辑者：可以编辑任务和项目信息

    - 查看者：只能查看，不能编辑

- ✅ **实时同步**：多人编辑时实时同步数据（v1.0 通过轮询实现）

- ✅ **操作日志**：记录所有任务的创建、修改、删除操作（数据库已预留）

### 4.2 辅助功能模块

#### 4.2.1 搜索与筛选

- ✅ **全局搜索**：搜索所有项目和任务

- ✅ **按条件筛选**：按负责人、优先级、状态、标签筛选任务

- ✅ **保存筛选**：保存常用的筛选条件

#### 4.2.2 数据导入导出

- ✅ **导出为JSON**：完整导出项目数据

- ✅ **导出为Markdown**：导出为大纲形式的Markdown文档

- ✅ **导出为Excel**：导出为表格形式的Excel文件

- ✅ **导入JSON**：从JSON文件导入项目数据

#### 4.2.3 提醒与通知

- ✅ **截止日期提醒**：任务截止前1天发送提醒

- ✅ **任务分配通知**：被分配任务时收到通知

- ✅ **评论通知**：被@或任务有新评论时收到通知

- ✅ **通知中心**：集中查看所有通知

#### 4.2.4 个人设置

- ✅ **个人信息**：修改头像、昵称、邮箱

- ✅ **密码修改**：修改登录密码

- ✅ **通知设置**：自定义哪些事件发送通知

- ✅ **界面设置**：切换浅色/深色主题

## 6. 技术架构建议

### 6.1 整体架构

采用前后端分离的跨平台应用架构：

- 前端：React + TypeScript + Vite（Web 应用）
- 后端：Python + FastAPI（Docker 容器化）
- 数据库：SQLite（异步）
- 实时通信：HTTP 轮询（v1.0），Socket.io（v2.0 规划）
- 部署：Docker + Docker Compose

### 6.2 前端技术栈

- **核心框架**：React 18 + TypeScript
- **构建工具**：Vite 5.x（开发热更新）
- **状态管理**：Zustand 4.x（轻量高效）
- **UI组件库**：Ant Design 5.x
- **拖拽库**：dnd-kit（任务拖拽）+ react-flow（依赖关系连线）
- **甘特图**：gantt-task-react（轻量开源）
- **富文本编辑器**：Tiptap 2.x（任务描述）
- **样式方案**：Tailwind CSS + CSS Modules
- **HTTP客户端**：Axios
- **路由**：React Router DOM 6.x

> 注：Tauri 集成已配置但桌面打包未执行

### 6.3 后端技术栈

- **运行环境**：Python 3.11 + FastAPI（Docker 容器）
- **Web框架**：FastAPI 0.109.x（现代化，自动生成 API 文档）
- **ORM**：SQLAlchemy 2.x（异步）
- **数据库**：SQLite（异步 aiosqlite）
- **身份认证**：JWT + passlib（bcrypt）
- **数据校验**：Pydantic 2.x
- **文件处理**：openpyxl（Excel 导出）
- **容器化**：Docker + Docker Compose
- **测试框架**：pytest + pytest-asyncio + httpx

> 注：Socket.io 实时通信待 v2.0 开发

### 6.5 API设计原则

- 采用RESTful API设计风格
- 所有接口返回统一的JSON格式 `{ code, data, message }`
- 使用HTTP状态码表示请求结果
- 接口版本控制（/api/v1/{tool-name}/）
- 所有接口需要身份验证（登录接口除外）

### 6.6 工具联动架构（扩展设计）

#### 6.6.1 设计目标
支持后续系列工具之间的互相调用和数据共享，形成工具生态系统。

#### 6.6.2 统一 API 规范
```
基础路径：/api/v1/{tool-name}/
例如：
  /api/v1/tasktree/projects/*    → TaskTree 任务树
  /api/v1/xxxtool/tasks/*       → 后续工具
```

#### 6.6.3 统一响应格式
```json
{
  "code": 200,           // 状态码
  "data": { ... },       // 数据
  "message": "success"  // 描述信息
}
```

#### 6.6.4 工具注册机制
- 新工具按规范开发后，注册到工具中心
- 工具中心记录：工具名称、API 前缀、描述

#### 6.6.5 数据共享策略
- 共享用户系统（统一认证）
- 可选共享数据库或独立数据库
- 工具间通过 HTTP API 互相调用

#### 6.6.6 后续工具接入流程
1. 新工具按规范开发 API
2. 注册到工具中心
3. 获取 Token 后可调用其他工具 API
4. 可选择将数据存入共享数据库
