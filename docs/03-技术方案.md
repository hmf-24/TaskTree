## 3. 核心模块设计

### 3.1 认证模块

```python
# JWT Token结构
{
  "sub": "user_id",
  "email": "user@example.com",
  "exp": 1712812800,
  "iat": 1710201600
}

# 密码加密：bcrypt
# Token有效期：7天（可配置）
```

### 3.2 项目管理模块

```
ProjectService
├── create_project()      # 创建项目
├── update_project()      # 更新项目
├── delete_project()      # 删除项目
├── list_projects()       # 项目列表
├── archive_project()     # 归档项目
└── get_project()         # 获取项目详情
```

### 3.3 任务树模块

```
TaskService
├── create_task()         # 创建任务
├── update_task()        # 更新任务
├── delete_task()        # 删除任务
├── move_task()          # 移动任务（调整层级/顺序）
├── get_task_tree()      # 获取任务树
├── batch_create()       # 批量创建
└── update_progress()    # 更新进度
```

**任务树数据结构**：
```typescript
interface TaskTreeNode {
  id: number;
  name: string;
  children: TaskTreeNode[];
  // ...其他属性
}
```

### 3.4 依赖关系模块

```
DependencyService
├── create_dependency()  # 创建依赖
├── delete_dependency()  # 删除依赖
├── check_dependency()   # 检查依赖（前置任务是否完成）
└── get_dependencies()   # 获取依赖关系
```

### 3.5 数据导入导出

```

├── export_json()        # 导出JSON
├── export_markdown()    # 导出Markdown
```

## 4. 实时通信设计

### 4.1 Socket.io 事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `task:created` | 服务端→客户端 | 任务创建通知 |
| `task:updated` | 服务端→客户端 | 任务更新通知 |
| `task:deleted` | 服务端→客户端 | 任务删除通知 |
| `task:moved` | 服务端→客户端 | 任务移动通知 |
| `project:updated` | 服务端→客户端 | 项目更新通知 |
| `member:joined` | 服务端→客户端 | 成员加入通知 |

### 4.2 房间设计

```
project:{project_id}    # 项目房间，所有成员加入
task:{task_id}          # 任务房间，相关人员加入
```

## 5. 文件存储设计

### 5.1 本地存储结构

```
backend/
└── uploads/
    └── {project_id}/
        └── {task_id}/
            └── {filename}
```

### 5.2 文件上传限制

- 单文件大小：< 10MB
- 支持类型：图片、文档、压缩包
- 文件名处理：UUID重命名，防止冲突

## 6. 部署方案

### 6.1 Docker Compose

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
      - ./uploads:/app/uploads
    environment:
      - DATABASE_URL=sqlite:///./data/tasktree.db

  frontend:
    build: ./frontend
    ports:
      - "3000:80"
```

### 6.2 Tauri 打包

```
构建命令：pnpm tauri build
输出：.exe / .dmg / .AppImage
```

## 7. 开发环境

### 7.1 本地开发

```bash
# 前端
cd frontend
pnpm install
pnpm dev

# 后端（Docker）
docker-compose up -d

# 或直接运行
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### 7.2 环境变量

```env
# backend/.env
DATABASE_URL=sqlite:///./data/tasktree.db
SECRET_KEY=your-secret-key
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=10080

# frontend/.env
VITE_API_BASE_URL=http://localhost:8000
```

## 8. 性能优化策略

| 场景 | 优化方案 |
|------|----------|
| 大量任务渲染 | 虚拟滚动（react-window） |
| 图片加载 | 懒加载 + 压缩 |
| API请求 | 请求缓存 + 乐观更新 |
| 实时同步 | 节流 + 增量更新 |
| 数据库查询 | 索引优化 + 懒加载 |
