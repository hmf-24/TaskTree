# 钉钉智能助手功能文档

## 功能概述

钉钉智能助手是 TaskTree 的核心功能之一，允许用户通过钉钉消息快速更新任务进度，系统会自动解析消息内容并更新相应的任务状态。

**版本**: 1.0.0  
**状态**: ✅ 已完成  
**完成日期**: 2024-04-24

---

## 核心功能

### 1. 用户绑定

用户可以将钉钉账号与 TaskTree 系统账号绑定，实现身份映射。

**功能特性**:
- 绑定钉钉账号
- 解除绑定
- 查询绑定状态
- 缓存机制（5 分钟 TTL）

**使用方法**:
1. 登录 TaskTree 系统
2. 进入"设置"页面
3. 点击"钉钉绑定"标签
4. 输入钉钉用户 ID 和昵称
5. 点击"绑定"按钮

### 2. 进度自动解析

系统支持自然语言进度反馈，自动识别进度类型和关键信息。

**支持的进度类型**:

| 类型 | 说明 | 示例 |
|------|------|------|
| completed | 任务完成 | "任务 A 完成了" |
| in_progress | 任务进行中 | "任务 B 进度 50%" |
| problem | 遇到问题 | "任务 C 遇到问题：API 接口返回 500 错误" |
| extend | 需要延期 | "任务 D 需要延期 3 天" |
| query | 查询状态 | "任务 E 的状态怎么样？" |

**解析机制**:
- **LLM 解析**（优先）：使用大语言模型理解自然语言
- **规则引擎降级**：当 LLM 不可用时，使用关键词识别

**提取信息**:
- 进度百分比（如"50%"、"百分之 60"）
- 问题描述（如"API 接口返回 500 错误"）
- 延期天数（如"延期 3 天"、"推迟 5 天"）
- 关键词（用于任务匹配）

### 3. 任务智能匹配

系统根据消息内容自动匹配相关任务。

**匹配算法**:
- **完全匹配**（权重 100）：任务名称完全匹配
- **模糊匹配**（权重 80/60）：任务名称或描述包含关键词
- **状态优先级**（+20）：进行中/待处理的任务优先
- **时间优先级**（+10）：最近更新的任务优先

**匹配结果**:
- 单个匹配：自动更新任务
- 多个匹配：返回任务列表供用户选择
- 无匹配：提示用户任务不存在

### 4. 任务自动更新

根据解析结果自动更新任务状态和信息。

**更新规则**:

| 进度类型 | 更新操作 |
|---------|---------|
| completed | 状态=已完成，进度=100% |
| in_progress | 状态=进行中，进度=N% |
| problem | 追加问题描述到任务描述 |
| extend | 截止日期延期 N 天 |
| query | 不修改任务（仅查询） |

**权限验证**:
- 只有任务所有者或项目成员可以更新任务
- 权限验证失败返回错误消息

**操作日志**:
- 记录更新来源为"钉钉反馈"
- 记录所有字段变更
- 记录时间戳

### 5. 进度反馈记录

系统记录所有进度反馈，支持历史查询。

**记录内容**:
- 原始消息内容
- 解析结果（JSON 格式）
- 反馈类型
- 创建时间

**查询功能**:
- 按任务 ID 筛选
- 分页支持（最多 50 条）
- 按时间倒序排列
- 权限验证

## 使用指南

### 配置钉钉机器人

1. **创建钉钉机器人**
   - 登录钉钉开放平台
   - 创建企业内部应用或机器人
   - 获取 Webhook URL 和 Secret

2. **配置 TaskTree**
   - 登录 TaskTree 系统
   - 进入"设置" → "钉钉配置"
   - 输入 Webhook URL 和 Secret
   - 点击"保存"

3. **测试连接**
   - 点击"发送测试消息"
   - 检查钉钉是否收到消息

### 绑定钉钉账号

1. 进入"设置" → "钉钉绑定"
2. 输入钉钉用户 ID（可在钉钉个人信息中查看）
3. 输入钉钉昵称
4. 点击"绑定"

### 发送进度反馈

在钉钉中向机器人发送消息，格式如下：

**完成任务**:
```
任务 A 完成了
完成了用户登录功能
```

**更新进度**:
```
任务 B 进度 50%
正在做数据库设计，已经完成 60%
```

**报告问题**:
```
任务 C 遇到问题：API 接口返回 500 错误
遇到 bug 了，无法连接数据库
```

**请求延期**:
```
任务 D 需要延期 3 天
来不及了，需要推迟 5 天
```

**查询状态**:
```
任务 E 的状态怎么样？
查询任务 F 的进度
```

### 查看反馈历史

1. 进入"设置" → "反馈历史"
2. 查看所有进度反馈记录
3. 可按任务筛选
4. 支持分页浏览

## API 接口

### 1. 绑定钉钉账号

**接口**: `POST /api/v1/dingtalk/bind`

**请求参数**:
```json
{
  "dingtalk_user_id": "string",
  "dingtalk_name": "string"
}
```

**响应**:
```json
{
  "message": "绑定成功",
  "data": {
    "user_id": 1,
    "dingtalk_user_id": "dingtalk_123",
    "dingtalk_name": "张三"
  }
}
```

### 2. 解除绑定

**接口**: `DELETE /api/v1/dingtalk/unbind`

**响应**:
```json
{
  "message": "解除绑定成功"
}
```

### 3. 查询绑定状态

**接口**: `GET /api/v1/dingtalk/binding`

**响应**:
```json
{
  "message": "success",
  "data": {
    "is_bound": true,
    "dingtalk_user_id": "dingtalk_123",
    "dingtalk_name": "张三"
  }
}
```

### 4. 钉钉消息回调

**接口**: `POST /api/v1/dingtalk/callback`

**请求头**:
- `x-dingtalk-timestamp`: 时间戳
- `x-dingtalk-sign`: 签名

**请求体**:
```json
{
  "senderId": "dingtalk_123",
  "text": {
    "content": "任务 A 完成了"
  }
}
```

**响应**:
```json
{
  "message": "success"
}
```

### 5. 查询进度反馈

**接口**: `GET /api/v1/progress-feedback`

**查询参数**:
- `task_id` (可选): 任务 ID
- `limit` (可选): 每页数量（默认 20，最大 50）
- `offset` (可选): 偏移量（默认 0）

**响应**:
```json
{
  "message": "success",
  "data": {
    "items": [
      {
        "id": 1,
        "user_id": 1,
        "task_id": 10,
        "message_content": "任务 A 完成了",
        "parsed_result": {
          "type": "completed",
          "progress": 100,
          "description": "",
          "extend_days": 0,
          "confidence": 0.95,
          "keywords": ["完成"]
        },
        "feedback_type": "completed",
        "created_at": "2024-04-24T10:00:00Z"
      }
    ],
    "total": 10,
    "limit": 20,
    "offset": 0
  }
}
```

### 6. 发送测试消息

**接口**: `POST /api/v1/dingtalk/test-message`

**请求参数**:
```json
{
  "message": "这是一条测试消息"
}
```

**响应**:
```json
{
  "message": "消息发送成功"
}
```

### 7. 健康检查

**接口**: `GET /api/v1/dingtalk/health`

**响应**:
```json
{
  "message": "success",
  "data": {
    "status": "healthy",
    "dingtalk_service": "ok",
    "llm_service": "ok"
  }
}
```

## 安全性

### 1. 签名验证

所有钉钉回调请求都需要验证签名，确保请求来自钉钉服务器。

**验证方法**: HMAC-SHA256

**验证流程**:
1. 提取请求头中的 `x-dingtalk-timestamp` 和 `x-dingtalk-sign`
2. 使用 Secret 计算签名
3. 比对计算结果和请求签名
4. 验证失败返回 401 错误

### 2. 时间戳验证

防止重放攻击，验证请求时间戳。

**验证规则**:
- 时间戳必须在 5 分钟窗口内
- 超时请求返回 401 错误

### 3. 频率限制

防止滥用，限制请求频率。

**限制规则**:
- 钉钉回调：10 次/分钟
- 绑定操作：5 次/分钟
- 测试消息：3 次/分钟

**超出限制**: 返回 429 错误

### 4. 权限验证

确保用户只能操作自己的任务。

**验证规则**:
- 任务所有者可以更新任务
- 项目成员可以更新任务
- 其他用户返回 403 错误

### 5. 安全日志

记录所有安全相关事件。

**记录内容**:
- 签名验证失败
- 时间戳过期
- 频率限制超出
- 权限拒绝
- 无效请求

## 常见问题

### Q: 如何获取钉钉用户 ID？
A: 在钉钉个人信息中查看，或联系管理员获取。

### Q: 为什么消息没有自动更新任务？
A: 可能的原因：
1. 钉钉账号未绑定
2. 任务名称不匹配
3. 没有任务操作权限
4. 消息格式不正确

### Q: 支持哪些消息格式？
A: 支持自然语言格式，如：
- "任务 A 完成了"
- "任务 B 进度 50%"
- "任务 C 遇到问题：XXX"
- "任务 D 需要延期 3 天"

### Q: 如何查看进度反馈历史？
A: 进入"设置" → "反馈历史"，可以查看所有反馈记录。

### Q: 系统如何匹配任务？
A: 系统使用智能匹配算法，根据任务名称、描述、关键词等多维度匹配。

### Q: 可以同时绑定多个钉钉账号吗？
A: 不可以，一个系统账号只能绑定一个钉钉账号。

### Q: 如何解除绑定？
A: 进入"设置" → "钉钉绑定"，点击"解除绑定"按钮。

## 技术架构

### 核心服务

1. **DingtalkUserMappingService** - 用户身份映射
2. **ProgressParserService** - 进度解析
3. **TaskMatcherService** - 任务匹配
4. **TaskUpdaterService** - 任务更新
5. **MessageParserService** - 消息解析
6. **MessagePrinterService** - 消息格式化
7. **AsyncTaskQueue** - 异步任务队列

### 数据模型

1. **UserNotificationSettings** - 用户通知设置（扩展）
   - dingtalk_user_id: 钉钉用户 ID
   - dingtalk_name: 钉钉用户昵称

2. **ProgressFeedback** - 进度反馈记录
   - user_id: 用户 ID
   - task_id: 任务 ID
   - message_content: 原始消息内容
   - parsed_result: 解析结果（JSON）
   - feedback_type: 反馈类型
   - created_at: 创建时间

### 消息处理流程

```
1. 钉钉用户发送消息
   ↓
2. POST /api/v1/dingtalk/callback 接收回调
   ├─ 验证签名（HMAC-SHA256）
   ├─ 验证时间戳（5 分钟窗口）
   ├─ 检查频率限制（10次/分钟）
   └─ 快速响应（< 200ms）
   ↓
3. 异步处理消息（后台任务队列）
   ├─ 用户身份映射（缓存查询）
   ├─ 进度解析（LLM + 规则引擎）
   ├─ 任务匹配（多维度匹配）
   └─ 任务更新（权限验证）
   ↓
4. 保存进度反馈记录
   ↓
5. 发送确认消息到钉钉
```
