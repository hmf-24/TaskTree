## 阶段八：Docker 部署实战与 Bug 清零
**日期**: 2026-04-23

### 🐛 P0 Bug 修复记录

| # | Bug 描述 | 根因 | 修复方案 |
|---|---------|------|---------|
| 1 | ESLint 依赖冲突导致 `npm install` 失败 | `@eslint/js ^10` 与 ESLint 9 不兼容 | 将 `@eslint/js` 降级为 `^9`；构建命令追加 `--legacy-peer-deps` |
| 2 | SQLite 路径错误，容器启动后数据库读写失败 | 相对路径 `sqlite+aiosqlite:///app/data/` 在容器内解析异常 | 改为绝对路径 `sqlite+aiosqlite:////app/data/` |
| 3 | `bcrypt 5.0.0` 引入 `bcrypt.check_password_hash` API 不兼容 | bcrypt 5.0 删除旧 API，`check_password_hash` 实现方式变更 | 添加 `bcrypt<5.0.0` 版本约束到 `requirements.txt` |
| 4 | `async_sessionmaker` 作为异步上下文管理器使用报错 | `AsyncSessionMaker` 不是 async context manager，直接当 `with` 使用会失败 | 改为 `get_session_maker()()`（先实例化再调用） |
| 5 | 日期选择器显示英文（应显示中文） | `main.tsx` 未引入 dayjs 中文语言包 | 在 `main.tsx` 中添加 `import 'dayjs/locale/zh-cn'` 并配置 `dayjs.locale('zh-cn')` |
| 6 | MiniMax API 返回 404 | MiniMax 已更换 API 端点，原 `api.minimax.chat` 已废弃 | 端点改为 `https://api.minimaxi.com/anthropic/v1/messages` |
| 7 | MiniMax API 返回 401 Unauthorized | 认证方式从 `x-api-key` Header 改为 `Authorization: Bearer` Token | 修改 `_call_minimax` 方法使用 `Bearer` 认证 |
| 8 | MiniMax 提供商名称前后端不一致，测试连通性始终返回 "Unknown provider" | 后端 `LLMService.__init__` 默认 `provider="minmax"` 且 API handler 只识别 `minimax`/`minmax`；前端传入 "minimax" 时无法匹配 | 在 `_call_api` 中增加 `if p in ("minimax", "minmax")` 双重匹配 |
| 9 | 分析维度配置（overdue、progress_stalled 等）保存后刷新丢失 | `setFieldsValue` 不支持嵌套路径（如 `analysis_config.overdue`）；同时后端 `create_or_update_settings` 未处理 `analysis_config` 字段 | 前端用独立 `useState(analysisConfig)` 管理开关状态并单独保存；后端增加 `settings.analysis_config = json.dumps(...)` 写入逻辑 |
| 10 | 前端页面构建后崩溃（`Settings.tsx` 存在重复变量声明） | 加载设置时两次重复声明 `const ac` | 删除重复声明行 |
