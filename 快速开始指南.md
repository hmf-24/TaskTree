# TaskTree 快速开始指南

## 🎉 新功能

### 1. 钉钉Stream模式客户端 ✨

现在支持在本地开发环境使用钉钉机器人，无需公网IP！

#### 配置步骤

1. **获取钉钉凭证**
   - 访问 [钉钉开放平台](https://open-dev.dingtalk.com/)
   - 创建企业内部应用
   - 获取 AppKey 和 AppSecret

2. **配置环境变量**
   
   编辑 `.env` 文件，添加：
   ```bash
   DINGTALK_STREAM_ENABLED=true
   DINGTALK_CLIENT_ID=你的AppKey
   DINGTALK_CLIENT_SECRET=你的AppSecret
   ```

3. **重启后端**
   ```bash
   docker-compose restart backend
   ```

4. **验证启动**
   ```bash
   docker-compose logs backend | grep "Stream"
   ```
   
   应该看到：
   ```
   🚀 正在启动钉钉Stream客户端...
   ✅ 钉钉Stream客户端已启动
   🎉 钉钉Stream模式已启动
   ```

#### 使用方法

在钉钉群聊中@机器人：

- **查询任务**: `@机器人 登录功能的详细情况`
- **更新进度**: `@机器人 登录功能完成了80%`
- **报告问题**: `@机器人 登录功能遇到了数据库问题`
- **延期任务**: `@机器人 登录功能需要延期3天`

### 2. 头像上传功能 📸

现在可以直接上传头像图片，无需手动输入URL！

#### 使用步骤

1. 登录系统
2. 进入"设置" → "个人资料"
3. 点击"上传图片"按钮
4. 选择图片文件（建议1:1比例）
5. 上传成功后自动填充URL
6. 点击"保存修改"

#### 支持格式

- ✅ JPG / JPEG
- ✅ PNG
- ✅ GIF
- ✅ WebP
- ✅ 其他图片格式

#### 建议

- 📐 使用1:1比例的图片（如 200x200, 400x400）
- 📦 文件大小建议 < 2MB
- 🎨 清晰的头像照片或图标

## 🐛 故障排查

### 钉钉机器人无响应

1. 检查Stream客户端是否启动：
   ```bash
   docker-compose logs backend | grep "Stream"
   ```

2. 检查环境变量配置：
   ```bash
   docker-compose exec backend env | grep DINGTALK
   ```

3. 确认用户已绑定钉钉账号（在系统设置中配置）

### 头像上传失败

1. 检查文件是否为图片格式
2. 检查文件大小是否超过限制（10MB）
3. 检查网络连接
4. 查看浏览器控制台错误信息

## 💡 提示

- Stream模式和Webhook模式可以共存
- 建议本地开发使用Stream模式
- 生产环境可以使用Webhook模式或Stream模式
- 头像URL也可以手动输入（如使用外部图床）
