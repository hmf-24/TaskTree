## 🎯 技术亮点

### Stream模式优势
1. **无需公网IP** - 通过WebSocket主动连接钉钉服务器

3. **自动重连** - SDK内置重连机制，网络中断后自动恢复

## 📝 使用说明

### 启用钉钉Stream模式

1. 编辑 `.env` 文件：
```bash
DINGTALK_STREAM_ENABLED=true
DINGTALK_CLIENT_ID=你的AppKey
DINGTALK_CLIENT_SECRET=你的AppSecret
```

2. 重启后端：
```bash
docker-compose restart backend
```

3. 查看日志确认启动：
```bash
docker-compose logs backend | grep "Stream"
```

### 使用头像上传

1. 登录系统
2. 进入"设置" → "个人资料"
3. 点击"上传图片"按钮
4. 选择图片文件
5. 等待上传完成
6. 点击"保存修改"

## 🎓 学习要点

### Stream模式关键概念
1. **主动连接** vs **被动接收**
2. **WebSocket长连接** vs **HTTP短连接**
3. **异步消息处理** vs **同步处理**
4. **自动重连机制**

### 前端上传最佳实践
