# 需求文档：钉钉Stream模式客户端

## 功能需求

### FR1: Stream客户端初始化

**描述**: 系统应能够在启动时初始化钉钉Stream客户端，建立WebSocket连接

**验收标准**:
- 系统从配置中读取 `DINGTALK_CLIENT_ID` 和 `DINGTALK_CLIENT_SECRET`
- 当 `DINGTALK_STREAM_ENABLED=true` 时，自动启动Stream客户端
- 成功建立与钉钉服务器的WebSocket连接
- 启动日志中记录"钉钉Stream客户端已启动"

**优先级**: P0（必需）

### FR2: 消息接收与分发

**描述**: 系统应能够实时接收钉钉推送的消息并分发到处理器

**验收标准**:
- 当用户在钉钉中@机器人发送消息时，系统能够接收到消息
- 消息包含必需字段：`senderId`（发送者ID）、`text.content`（消息内容）
- 消息被正确分发到 `DingtalkMessageHandler.handle_message()`
- 接收到消息后在200ms内返回响应

**优先级**: P0（必需）

### FR3: 用户身份映射

**描述**: 系统应能够将钉钉用户ID映射到系统用户

**验收标准**:
- 根据 `senderId` 查询 `UserNotificationSettings` 表
- 如果找到映射，获取对应的 `user_id`
- 如果未找到映射，发送绑定引导消息
- 查询使用数据库索引，响应时间 < 50ms

**优先级**: P0（必需）

### FR4: 消息处理复用

**描述**: 系统应复用现有的 `process_dingtalk_message` 函数处理消息

**验收标准**:
- 调用 `app.api.v1.dingtalk.process_dingtalk_message` 处理消息
- 传递参数：`user_id`, `dingtalk_user_id`, `message_content`, `db`
- 处理逻辑包括：进度解析、任务匹配、状态更新、响应发送
- 处理过程在后台异步执行，不阻塞消息响应

**优先级**: P0（必需）

### FR5: 频率限制

**描述**: 系统应对用户消息进行频率限制，防止滥用

**验收标准**:
- 使用现有的 `dingtalk_rate_limiter` 检查频率
- 限制规则：每用户每分钟最多10条消息
- 超出限制时，发送友好提示消息
- 超出限制时，仍返回成功响应（避免钉钉重试）

**优先级**: P1（重要）

### FR6: 生命周期管理

**描述**: 系统应正确管理Stream客户端的生命周期

**验收标准**:
- 应用启动时，在 `lifespan` 中启动Stream客户端
- 应用关闭时，优雅地停止Stream客户端
- 停止时关闭WebSocket连接并清理资源
- 关闭日志中记录"钉钉Stream客户端已停止"

**优先级**: P0（必需）

### FR7: 配置管理

**描述**: 系统应支持通过环境变量配置Stream模式

**验收标准**:
- 在 `app.core.config.Settings` 中添加配置项
- 支持配置项：`DINGTALK_CLIENT_ID`, `DINGTALK_CLIENT_SECRET`, `DINGTALK_STREAM_ENABLED`
- 配置可通过 `.env` 文件或环境变量设置
- 配置缺失时，记录警告并降级到Webhook模式

**优先级**: P0（必需）

### FR8: 错误处理与恢复

**描述**: 系统应能够处理各种错误情况并自动恢复

**验收标准**:
- 连接失败时，SDK自动重试（指数退避）
- 消息处理异常时，捕获并记录错误，不影响后续消息
- 数据库异常时，返回错误提示给用户
- 所有错误都返回成功响应给钉钉（避免重试）

**优先级**: P1（重要）

## 非功能需求

### NFR1: 性能要求

**描述**: 系统应满足性能指标

**验收标准**:
- 消息响应时间 < 200ms（P95）
- 用户身份查询时间 < 50ms
- 支持并发处理多个用户消息
- WebSocket连接稳定，无频繁断连

**优先级**: P0（必需）

### NFR2: 可靠性要求

**描述**: 系统应保证消息处理的可靠性

**验收标准**:
- 消息不丢失：所有推送的消息都被处理
- 连接断开后自动重连
- 异常情况下不影响应用主流程
- 消息处理幂等性（重复消息不产生副作用）

**优先级**: P0（必需）

### NFR3: 安全要求

**描述**: 系统应保证通信和数据安全

**验收标准**:
- WebSocket使用TLS加密（wss://）
- `client_secret` 不硬编码，通过环境变量配置
- 日志中不记录敏感信息
- 复用现有的频率限制和安全日志机制

**优先级**: P0（必需）

### NFR4: 可维护性要求

**描述**: 系统应易于维护和调试

**验收标准**:
- 代码结构清晰，职责分离
- 关键操作有日志记录
- 配置项有清晰的注释说明
- 错误信息详细且易于理解

**优先级**: P1（重要）

### NFR5: 兼容性要求

**描述**: 系统应与现有功能兼容

**验收标准**:
- 不影响现有Webhook模式的功能
- 复用现有的服务层代码（`process_dingtalk_message`）
- 复用现有的频率限制、缓存、安全日志机制
- 可通过配置在Stream模式和Webhook模式间切换

**优先级**: P0（必需）

## 约束条件

### C1: 技术栈约束

- 必须使用 `dingtalk-stream` SDK（版本 1.2.0）
- 必须使用 Python 3.8+
- 必须基于 FastAPI 框架
- 必须使用 SQLAlchemy 异步ORM

### C2: 部署环境约束

- 支持本地开发环境（无公网IP）
- 支持Docker容器化部署
- 需要稳定的网络连接（访问钉钉服务器）

### C3: 钉钉平台约束

- 必须使用钉钉开放平台注册的应用
- 必须配置正确的 `AppKey` 和 `AppSecret`
- 机器人类型必须是Stream模式（不是Webhook模式）

## 验收测试

### AT1: 基本消息接收测试

**步骤**:
1. 配置 `DINGTALK_STREAM_ENABLED=true` 和有效的凭证
2. 启动应用
3. 在钉钉中@机器人发送消息："测试消息"

**预期结果**:
- 应用日志显示"钉钉Stream客户端已启动"
- 应用接收到消息并处理
- 机器人回复相应消息

### AT2: 用户绑定流程测试

**步骤**:
1. 使用未绑定的钉钉账号@机器人
2. 发送任意消息

**预期结果**:
- 机器人回复："请先绑定钉钉账号，访问系统设置进行绑定"

### AT3: 任务更新测试

**步骤**:
1. 使用已绑定的账号@机器人
2. 发送消息："登录功能完成了80%"

**预期结果**:
- 系统解析进度为80%
- 匹配到"登录功能"任务
- 更新任务进度为80%
- 机器人回复确认消息

### AT4: 频率限制测试

**步骤**:
1. 在1分钟内连续发送超过10条消息

**预期结果**:
- 前10条消息正常处理
- 第11条消息返回频率限制提示
- 等待1分钟后恢复正常

### AT5: 连接恢复测试

**步骤**:
1. 启动应用，建立连接
2. 断开网络连接30秒
3. 恢复网络连接

**预期结果**:
- 断网期间，SDK记录连接失败日志
- 恢复网络后，SDK自动重连成功
- 重连后消息接收恢复正常

### AT6: 应用关闭测试

**步骤**:
1. 启动应用
2. 发送SIGTERM信号关闭应用

**预期结果**:
- 应用优雅关闭
- 日志显示"钉钉Stream客户端已停止"
- WebSocket连接正确关闭

## 依赖关系

### 上游依赖

- 钉钉开放平台：提供Stream模式API和SDK
- 钉钉应用配置：需要正确的AppKey和AppSecret

### 下游依赖

- 现有消息处理逻辑：`process_dingtalk_message`
- 现有服务层：`dingtalk_service`, `llm_service`, `progress_parser_service`
- 数据库：`UserNotificationSettings` 表

## 风险与缓解

### 风险1: SDK稳定性

**描述**: `dingtalk-stream` SDK可能存在bug或不稳定
**影响**: 消息接收失败或连接频繁断开
**缓解**: 
- 使用稳定版本（1.2.0）
- 添加详细的错误日志
- 保留Webhook模式作为备选方案

### 风险2: 网络依赖

**描述**: 本地网络不稳定可能导致连接中断
**影响**: 消息接收延迟或丢失
**缓解**:
- SDK内置自动重连机制
- 添加连接状态监控
- 记录连接异常日志

### 风险3: 性能瓶颈

**描述**: 高并发消息可能导致处理延迟
**影响**: 响应时间超过200ms
**缓解**:
- 使用后台任务队列异步处理
- 数据库查询优化（索引）
- 添加性能监控

### 风险4: 配置错误

**描述**: 用户可能配置错误的凭证
**影响**: 无法建立连接
**缓解**:
- 启动时验证配置
- 提供清晰的错误提示
- 文档中提供配置示例
