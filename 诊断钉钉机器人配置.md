# 诊断钉钉机器人配置

## 问题

OpenClaw 使用同一个机器人可以交互，但 TaskTree 不行。

## 可能原因

### 1. 回调地址指向了 OpenClaw

**症状**：
- @机器人后，OpenClaw 能收到回调
- TaskTree 收不到回调

**解决**：
- 修改机器人回调地址指向 TaskTree
- 或创建新的机器人给 TaskTree 使用

### 2. 机器人配置了多个回调地址

某些钉钉机器人支持配置多个回调地址。

**解决**：
- 添加 TaskTree 的回调地址到列表中

### 3. 消息路由问题

OpenClaw 可能在回调处理中做了路由。

---

## 诊断步骤

### 步骤1：查看机器人配置

1. 打开钉钉群
2. 找到机器人
3. 点击机器人头像或设置
4. 查看配置信息：
   - 机器人类型
   - 回调地址（POST URL）
   - Token
   - Secret

### 步骤2：检查回调地址

**当前回调地址是什么？**

- [ ] 指向 OpenClaw 的服务器
- [ ] 指向 TaskTree 的服务器
- [ ] 其他地址

### 步骤3：测试回调

使用测试脚本验证 TaskTree 的回调接口是否正常：

```bash
# 在 TaskTree 项目目录下
python test_dingtalk_callback.py
```

如果返回 200 OK，说明接口正常。

---

## 解决方案

### 方案A：修改现有机器人（推荐）

**适用场景**：你有机器人管理权限

**步骤**：

1. **编辑机器人**：
   - 在钉钉群中找到机器人
   - 点击"编辑"或"设置"

2. **修改回调地址**：
   - 旧地址：`https://openclaw-server.com/callback`
   - 新地址：`https://your-tasktree-server.com/api/v1/dingtalk/callback`

3. **保存配置**

4. **在 TaskTree 中配置**：
   - Webhook URL：机器人的发送地址
   - Secret：机器人的加签密钥

5. **测试**：
   - @机器人发送消息
   - 查看 TaskTree 后端日志

### 方案B：创建新机器人

**适用场景**：无法修改现有机器人，或想保留 OpenClaw 的功能

**步骤**：

1. **创建新的 Outgoing 机器人**：
   - 群设置 → 智能群助手 → 添加机器人
   - 选择"自定义" → "Outgoing机器人"
   - 名称：TaskTree 助手

2. **配置回调地址**：
   - 如果本地开发：使用 ngrok
     ```bash
     ngrok http 8000
     # 得到：https://abc123.ngrok.io
     ```
   - 回调地址：`https://abc123.ngrok.io/api/v1/dingtalk/callback`

3. **记录配置**：
   - Token：`xxxxxx`
   - Secret：`SECxxxxxx`

4. **在 TaskTree 中配置**：
   - Webhook URL：`https://oapi.dingtalk.com/robot/send?access_token=你的token`
   - Secret：`SECxxxxxx`

5. **测试**：
   - @TaskTree助手 发送消息
   - 应该能收到回复

### 方案C：使用 ngrok 让本地服务可访问

**适用场景**：机器人已经是 Outgoing 类型，只需要让本地服务可访问

**步骤**：

1. **启动 ngrok**：
   ```bash
   ngrok http 8000
   ```

2. **获取公网地址**：
   ```
   Forwarding: https://abc123.ngrok.io -> http://localhost:8000
   ```

3. **修改机器人回调地址**：
   ```
   https://abc123.ngrok.io/api/v1/dingtalk/callback
   ```

4. **测试**：
   - @机器人发送消息
   - 查看 TaskTree 日志是否收到回调

## 常见问题

### Q: 如何知道机器人是 Webhook 还是 Outgoing？

**A**: 
- Webhook 机器人：只有"Webhook地址"，没有"回调地址"
- Outgoing 机器人：有"回调地址（POST URL）"配置

### Q: 可以同时使用两个机器人吗？

**A**: 
- 可以！一个群可以添加多个机器人
- OpenClaw 用一个，TaskTree 用另一个
- 互不干扰

### Q: ngrok 免费版够用吗？

**A**:
- 测试够用
- 限制：每次启动 URL 会变（需要重新配置机器人）
- 生产环境建议部署到公网服务器

### Q: 如何查看机器人配置？

**A**:
1. 在钉钉群中找到机器人
2. 长按机器人消息 → 查看机器人信息
3. 或者：群设置 → 智能群助手 → 找到机器人 → 查看详情
