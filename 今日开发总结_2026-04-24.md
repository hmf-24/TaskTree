## 二、Docker部署问题修复 ✅

### 2.1 问题描述

Docker容器重启时出现数据库表重复创建错误：
```
table progress_feedback already exists
```

### 2.2 根本原因

- SQLAlchemy的 `Base.metadata.create_all()` 在某些情况下会尝试创建已存在的表
- 数据库初始化逻辑缺少异常处理

### 2.3 解决方案

**修改文件**: `backend/app/core/database.py`

在 `init_db()` 函数中添加异常处理：
```python
async def init_db():
    """应用启动时初始化数据库：创建所有未存在的表结构。"""
    from app.models import Base
    engine = get_engine()
    try:
        async with engine.begin() as conn:
            await conn.run_sync(Base.metadata.create_all)
        print("✓ 数据库初始化成功")
    except Exception as e:
        # 如果表已存在，忽略错误继续启动
        print(f"⚠ 数据库初始化警告: {e}")
        print("  继续启动应用...")
```
