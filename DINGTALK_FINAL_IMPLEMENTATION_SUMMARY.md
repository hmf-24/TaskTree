## 🎯 核心功能流程

### 完整的消息处理流程

```
1. 钉钉用户发送消息
   ↓
2. POST /api/v1/dingtalk/callback 接收回调
   ├─ 验证签名（HMAC-SHA256）
   ├─ 验证时间戳（5 分钟窗口）
   ├─ 检查频率限制（10次/分钟）
   └─ 快速响应（< 200ms）
   ↓
3. 异步处理消息（后台任务队列）
   ├─ 用户身份映射（DingtalkUserMappingService）
   │   └─ 缓存查询（5 分钟 TTL）
   ├─ 进度解析（ProgressParserService）
   │   ├─ LLM 解析（优先）
   │   └─ 规则引擎降级（LLM 失败时）
   ├─ 任务匹配（TaskMatcherService）
   │   ├─ 完全匹配（权重 100）
   │   ├─ 模糊匹配（权重 80/60）
   │   ├─ 状态优先级（+20）
   │   └─ 时间优先级（+10）
   └─ 任务更新（TaskUpdaterService）
       ├─ 权限验证
       ├─ 状态转换
       ├─ 进度更新
       ├─ 操作日志
       └─ 反馈保存
   ↓
4. 发送确认消息到钉钉（MessagePrinterService）
   ├─ 格式化任务信息
   ├─ 格式化进度条
   └─ 格式化状态标签
```

## 📝 使用指南

### 1. 后端部署

```bash
# 1. 运行数据库迁移
cd backend
python migrations/create_progress_feedback.py

# 2. 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 2. 前端部署

```bash
# 1. 安装依赖
cd frontend
npm install

# 2. 构建
npm run build

# 3. 部署
cp -r dist/* /var/www/html/
```

### 3. 钉钉配置

1. 创建钉钉机器人
2. 获取 Webhook URL 和 Secret
3. 在设置页面配置 Webhook 和 Secret
