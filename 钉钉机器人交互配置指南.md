# 钉钉机器人交互功能配置指南

## 前提条件

- TaskTree 服务已启动（Docker 或本地）
- 有钉钉群管理员权限
- 需要公网可访问的回调地址

---

## 方案1：使用 ngrok 内网穿透（本地开发推荐）

### 步骤1：安装 ngrok

1. 访问 https://ngrok.com/download
2. 下载对应系统的版本（Windows/Mac/Linux）
3. 解压到任意目录

### 步骤2：启动 ngrok

在命令行中运行：

```bash
# Windows
ngrok.exe http 8000

# Mac/Linux
./ngrok http 8000
```

你会看到类似输出：

```
ngrok by @inconshreveable

Session Status                online
Account                       your-account (Plan: Free)
Version                       3.0.0
Region                        United States (us)
Forwarding                    https://abc123.ngrok.io -> http://localhost:8000
```

**记录这个公网地址**：`https://abc123.ngrok.io`

### 步骤3：配置钉钉 Outgoing 机器人

1. **打开钉钉群** → 群设置 → 智能群助手

2. **添加机器人** → 自定义 → 通过Webhook接入

3. **选择机器人类型**：
   - ⚠️ 注意：要选择"Outgoing机器人"，不是普通的Webhook机器人

4. **配置机器人信息**：
   - 机器人名称：TaskTree 助手
   - 回调地址（POST）：`https://abc123.ngrok.io/api/v1/dingtalk/callback`
   - Token：系统自动生成（记录下来）

5. **配置安全设置**：
   - 选择"加签"
   - 记录 Secret（类似：SECxxxxxxxxxxxx）

6. **完成添加**

### 步骤4：在 TaskTree 中配置

1. 登录 TaskTree
2. 进入"设置"页面
3. 找到"智能提醒"部分
4. 配置：
   - **钉钉 Webhook 地址**：`https://oapi.dingtalk.com/robot/send?access_token=你的token`
   - **钉钉密钥**：`SECxxxxxxxxxxxx`（你的Secret）
   - **大模型配置**：配置 LLM API Key
5. 保存设置

### 步骤5：绑定钉钉用户

1. 在钉钉中找到你的用户ID：
   - 打开钉钉 → 我的 → 设置 → 我的信息
   - 复制"钉钉号"或"用户ID"

2. 在 TaskTree 设置中绑定：
   - 钉钉用户ID：你的钉钉号
   - 钉钉昵称：你的昵称

### 步骤6：测试

1. **测试主动提醒**：
   - 点击"立即提醒"按钮
   - 10-15秒后钉钉收到消息 ✅

2. **测试交互功能**：
   - 在钉钉群中 @TaskTree助手
   - 发送："任务1的详细情况"
   - 机器人应该回复任务详情 ✅

## 方案2：部署到公网服务器（生产环境）

### 步骤1：准备服务器

- 购买云服务器（阿里云、腾讯云等）
- 配置域名（可选但推荐）
- 安装 Docker 和 Docker Compose

### 步骤2：部署 TaskTree

```bash
# 克隆代码
git clone https://github.com/your-repo/TaskTree.git
cd TaskTree

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 启动服务
docker-compose -f docker-compose.prod.yml up -d
```

### 步骤3：配置 Nginx 和 SSL

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/v1/dingtalk/callback {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 步骤4：配置钉钉机器人

回调地址：`https://yourdomain.com/api/v1/dingtalk/callback`

其他步骤同方案1。

## 方案3：仅使用主动提醒（无需配置）

如果暂时不需要交互功能，可以只使用主动提醒：

### 当前可用功能

1. **立即提醒**：
   - 点击"立即提醒"按钮
   - 系统分析任务并发送钉钉消息

2. **定时提醒**（需要启动调度器）：
   - 系统每30分钟自动检查
   - 发现需要提醒的任务自动发送

### 配置步骤

1. 只需配置 Webhook 机器人（当前已配置）
2. 配置 LLM API Key
3. 点击"立即提醒"即可使用

## 常见问题

### Q1: ngrok 连接不稳定怎么办？

**A**: 
- 使用付费版 ngrok（更稳定）
- 使用国内的内网穿透工具（如 natapp、花生壳）
- 部署到公网服务器

### Q2: 机器人没有回复怎么办？

**A**: 检查以下几点：
1. 确认使用的是 Outgoing 机器人（不是普通 Webhook）
2. 确认回调地址可以从公网访问
3. 查看后端日志是否收到回调请求
4. 确认已绑定钉钉用户ID

### Q3: 如何查看后端日志？

**A**:
```bash
# Docker 部署
docker-compose logs backend --tail=100

# 查看实时日志
docker-compose logs -f backend
```

### Q4: 回调地址配置错误怎么办？

**A**:
1. 删除旧的机器人
2. 重新添加 Outgoing 机器人
3. 配置正确的回调地址

### Q5: 免费版 ngrok 有什么限制？

**A**:
- 每次启动 URL 会变化（需要重新配置机器人）
- 有连接数限制
- 建议：测试完成后部署到公网服务器
