## 部署指南

### 环境要求

- Python 3.8+
- Node.js 16+
- PostgreSQL/MySQL
- Redis（可选，用于生产环境）

### 后端部署

```bash
# 1. 安装依赖
cd backend
pip install -r requirements.txt

# 2. 运行数据库迁移
python migrations/add_dingtalk_fields.py
python migrations/create_progress_feedback.py

# 3. 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 前端部署

```bash
# 1. 安装依赖
cd frontend
npm install

# 2. 构建
npm run build

# 3. 部署到 Nginx
cp -r dist/* /var/www/html/
```

### 钉钉配置

1. 创建钉钉机器人
2. 获取 Webhook URL 和 Secret
3. 配置回调地址: `https://your-domain.com/api/v1/dingtalk/callback`
4. 在用户设置中绑定钉钉账号

## 使用指南

### 用户绑定流程

1. 登录 TaskTree 系统
2. 进入"设置"页面
3. 点击"钉钉绑定"标签
4. 点击"绑定钉钉账号"按钮
5. 输入钉钉用户 ID 和昵称
6. 点击"确认绑定"

### 发送进度反馈

在钉钉中发送消息：

```
# 完成任务
完成任务 A

# 更新进度
任务 B 进行中，进度 50%

# 报告问题
任务 C 遇到问题：需要更多资源

# 请求延期
任务 D 需要延期 3 天
```

### 测试消息

1. 在设置页面点击"发送测试消息"
2. 输入测试消息内容
3. 点击"发送"
4. 在钉钉中查看收到的消息

## 已知限制

### 技术限制

1. **内存缓存**
   - 不支持分布式部署
   - 重启后缓存丢失
   - 建议生产环境使用 Redis

2. **同步处理**
   - 消息处理目前是同步的
   - 建议集成 Celery/RQ

3. **LLM 依赖**
   - 依赖外部 LLM 服务
   - 有规则引擎降级方案

### 功能限制

1. **单轮对话**
   - 目前不支持多轮对话
   - 每次消息独立处理

2. **任务匹配**
   - 基于关键词匹配
   - 可能存在歧义
